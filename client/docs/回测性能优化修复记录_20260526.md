## 二、根因分析

### 2.1 进程池死锁（executor.rs）

`sandbox/executor.rs` 中的 `send_request` 函数存在死锁问题：

**问题代码：**
```rust
fn send_request(&mut self, input: &serde_json::Value, _timeout_secs: u64) -> Result<SandboxResult, SandboxError> {
    let stdin = self.child.stdin.as_mut()...;
    serde_json::to_writer(&mut *stdin, input)...;
    stdin.write_all(b"\n")...;
    stdin.flush()...;
    // 缺陷：stdin 未 drop，Python 收不到 EOF，一直等待输入
    let stdout = self.child.stdout.as_mut()...;
    // ... 读取 stdout
}
```

**对比 fallback 函数（正确做法）：**
```rust
let mut writer = stdin;
serde_json::to_writer(&mut writer, input)...;
writer.write_all(b"\n")...;
drop(writer);  // 关键：drop 让 Python 收到 EOF
```

### 2.2 sandbox_runner.py 路径错误

`executor.rs` 中路径计算逻辑：
```rust
let runner_path = std::env::current_exe()
    .and_then(|p| p.parent().and_then(|p| p.parent()).and_then(|p| p.parent()))
    .map(|p| p.join("sandbox_runner.py"));
```

二进制文件位置：`/home/hong/ACDA-Q-RUST-v1/server-rust/target/release/acda-q-server`

往上 3 层后指向：`/home/hong/ACDA-Q-RUST-v1/`（项目根目录）

但 `sandbox_runner.py` 实际位于：`/home/hong/ACDA-Q-RUST-v1/server-rust/sandbox_runner.py`

### 2.3 VITE_API_BASE 重复前缀

客户端 `.env` 文件配置：
```
VITE_API_BASE=http://100.68.25.78:8000/api/v1
```

前端代码拼接 WebSocket URL：
```typescript
const wsUrl = `${apiBase.replace(/^http/, "ws")}/api/v1/backtests/${id}/ws`;
// 结果：ws://100.68.25.78:8000/api/v1/api/v1/backtests/xxx/ws  （重复）
```

## 三、修复内容

### 3.1 修复进程池死锁

**文件：** `/home/hong/ACDA-Q-RUST-v1/server-rust/src/sandbox/executor.rs`

**修改：** 在 `send_request` 函数中添加 `drop(stdin)` 确保 Python 进程能收到 EOF：

```rust
fn send_request(&mut self, input: &serde_json::Value, _timeout_secs: u64) -> Result<SandboxResult, SandboxError> {
    let result = {
        let stdin = self.child.stdin.as_mut()...;
        serde_json::to_writer(&mut *stdin, input)...;
        stdin.write_all(b"\n")...;
        stdin.flush()...;
        drop(stdin);  // 关键修复：让 Python 收到 EOF

        let stdout = self.child.stdout.as_mut()...;
        // ... 读取逻辑
    };
    result
}
```
